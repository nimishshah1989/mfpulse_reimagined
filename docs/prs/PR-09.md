# PR-09: Sector Intelligence — Compass Bridge + Fund Drill-Down

## Objective
Build the Sector Intelligence page that bridges MarketPulse sector compass data into mutual fund recommendations. When a sector is "Leading" on the compass, this page shows the best MF vehicles to capture that momentum. This is the unique competitive edge — no other platform connects real-time sector momentum to fund selection.

---

## APIs Consumed

- `GET /api/v1/marketpulse/sectors?period=3M` — sector RS scores + quadrant classification
- `GET /api/v1/marketpulse/breadth?lookback=1y` — breadth history for timeline
- `GET /api/v1/marketpulse/sentiment` — sentiment score for context
- `GET /api/v1/marketpulse/regime` — market regime (risk-on/risk-off)
- `GET /api/v1/funds?category=&sort_by=return_1y` — fund list (used for sector-heavy funds)
- `GET /api/v1/funds/{mstar_id}/sectors` — per-fund sector exposure
- `GET /api/v1/lens/scores?category=&limit=500` — lens scores for filtered funds

**Note:** If MarketPulse is unavailable (port 8000 not running), show a clear "MarketPulse offline — sector data unavailable" message with the option to use cached/last-known data. Never crash.

### File 2: `web/src/components/sectors/CompassChart.jsx`

D3.js four-quadrant scatter chart.

**Design:**
- X-axis: RS Score (0–100). Y-axis: RS Momentum (change in RS score)
- Quadrant lines at x=50, y=0 creating 4 zones:
  - Top-right: **Leading** (high RS, improving) — green background tint
  - Top-left: **Improving** (low RS, improving) — teal background tint
  - Bottom-right: **Weakening** (high RS, declining) — amber background tint
  - Bottom-left: **Lagging** (low RS, declining) — red background tint
- Each sector = a circle at (RS_score, RS_momentum). Size = number of funds with >10% exposure to that sector. Color = quadrant color.
- Label: sector name next to each circle (collision-free placement using D3 force)
- **Click sector** → updates the fund drill-down below
- **Hover sector** → tooltip: sector name, RS score, momentum, quadrant, # funds

**Historical trail:** Each sector shows a faded trail of its position 1M ago and 3M ago (2 smaller, fainter dots connected by a line). This shows sector trajectory — is Technology moving from Improving to Leading?

**Responsiveness:** Min 400×400px, scales to available width.

### File 4: `web/src/components/sectors/FundDrillDown.jsx`

Fund recommendations for the selected sector. Appears when user clicks a sector on the compass.

**How it works:**
1. Fetch all funds with sector exposure data
2. Filter to funds where selected sector exposure > 10%
3. Sort by blended quality score (sector_exposure × average_lens_score)
4. Display as ranked cards

**Each fund card shows:**
- Rank number
- Fund name + AMC
- Sector exposure % (with progress bar)
- 1Y return (green/red)
- Mini 6-lens radar (tiny version, 60×60px, no labels — just the shape)
- Return class badge (LEADER/STRONG/AVERAGE/WEAK)
- Two action buttons: "Deep Dive →" and "Simulate →"

**Sort controls:** Sort by Exposure %, by Return 1Y, by Return Score, by Alpha Score, by Risk Score

**Filter:** Category filter (Large Cap, Mid Cap, Small Cap, Flexi Cap, Sectoral, Thematic) — because a "Sectoral Technology" fund has 92% exposure but a "Flexi Cap" might have 25% in Tech and be more diversified.

### File 5: `web/src/components/sectors/RotationTimeline.jsx`

Horizontal swim-lane chart showing sector quadrant positions over 24 months.

**Design:**
- Rows: one per sector (all 11 Morningstar sectors)
- Columns: months (left = 24M ago, right = current)
- Each cell colored by quadrant: green (Leading), teal (Improving), amber (Weakening), red (Lagging), gray (no data)

**Data source:** This requires MarketPulse to have historical sector data. If not available, show "Historical data coming soon" placeholder. The chart uses `fetchSectors()` for current data; historical may need a new MarketPulse endpoint or cached snapshots.

## Key Design Decision: Sector Exposure Data

Currently, we don't have portfolio/holdings data from Morningstar (the API for sector exposure `EquitySectorTechnologyNet` etc. hasn't been tested in the bulk API yet). The drill-down needs sector exposure percentages per fund.

**Phase 1 (this PR):** If sector exposure data is available in the DB, use it. If not, the drill-down section shows a "Sector exposure data loading — check back after next data refresh" message. The compass itself works regardless (it's pure MarketPulse data).

**Phase 2 (after portal edit):** Once the Portfolio Data API is configured on Morningstar portal and the fetcher pulls sector exposure, the drill-down auto-populates.

**Fallback approach:** If no sector exposure at fund level, use category-based mapping (Large Cap funds = primarily Financial Services + Technology, Pharma funds = 90%+ Healthcare, etc.). This is approximate but directionally correct.
