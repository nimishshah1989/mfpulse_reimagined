# PR-01: Morningstar Feed Ingestion Pipeline + MarketPulse Client

## Objective
Build the data ingestion layer that populates the database from Morningstar CSV feeds and the HTTP client that bridges to MarketPulse. At the end of this PR, you can drop a Morningstar CSV file into a directory and have it parsed, validated, and bulk-upserted into the correct tables — with full audit trail logging.

---

## Context

Morningstar delivers data via the Feed Builder as CSV files. Each feed type has different columns that must map to our database schema. The ingestion pipeline needs to:
1. Parse CSV with correct column mapping (Morningstar field names → our DB column names)
2. Validate data types (dates, decimals, booleans)
3. Bulk upsert (insert new, update existing — keyed by mstar_id + date)
4. Log every ingestion run to `ingestion_log` and `audit_trail`
5. Handle partial failures gracefully (one bad row doesn't kill the batch)

MarketPulse is already running on the same box (localhost:8000). MF Pulse needs a thin HTTP client to fetch breadth, sentiment, and sector data.

### File 2: `backend/app/ingestion/feed_parser.py`

Core CSV parsing engine. One method per feed type.

```python
class FeedParser:
    """
    Parses Morningstar CSV feeds into lists of dicts ready for DB insertion.
    Handles: column mapping, type coercion, validation, error collection.
    """

    def parse_master_feed(self, csv_path: str) -> ParseResult
    def parse_nav_feed(self, csv_path: str) -> ParseResult
    def parse_risk_stats_feed(self, csv_path: str) -> ParseResult
    def parse_rank_feed(self, csv_path: str) -> ParseResult
    def parse_holdings_feed(self, csv_path: str) -> ParseResult
    def parse_category_returns_feed(self, csv_path: str) -> ParseResult
```

Each method returns:
```python
@dataclass
class ParseResult:
    records: list[dict]        # Successfully parsed rows
    errors: list[dict]         # Rows that failed parsing (with row number + error)
    total_rows: int
    parsed_rows: int
    feed_type: str
    source_file: str
```

Type coercion rules:
- Dates: Accept `YYYY-MM-DD`, `DD-MM-YYYY`, `DD/MM/YYYY`. Parse to `datetime.date`.
- Decimals: Parse to `Decimal`. Strip commas, handle empty/null/NaN → `None`.
- Booleans: Accept `true/false`, `1/0`, `yes/no`, `True/False` → Python `bool`.
- Integers: Parse to `int`. Empty → `None`.
- Strings: Strip whitespace. Empty string → `None`.

Every row must have `mstar_id` (or the equivalent key field). Rows without it are added to `errors`.

### File 3: `backend/app/repositories/ingestion_repo.py`

Bulk upsert operations for each table.

```python
class IngestionRepository:
    """
    Bulk upsert operations for Morningstar feed data.
    Uses PostgreSQL INSERT ... ON CONFLICT for atomic upserts.
    """

    def __init__(self, db: Session): ...

    def upsert_fund_masters(self, records: list[dict]) -> UpsertResult
    def upsert_nav_daily(self, records: list[dict]) -> UpsertResult
    def upsert_risk_stats(self, records: list[dict]) -> UpsertResult
    def upsert_ranks(self, records: list[dict]) -> UpsertResult
    def upsert_holdings_snapshot(self, records: list[dict]) -> UpsertResult
    def upsert_holding_details(self, snapshot_id: UUID, records: list[dict]) -> UpsertResult
    def upsert_sector_exposure(self, records: list[dict]) -> UpsertResult
    def upsert_asset_allocation(self, records: list[dict]) -> UpsertResult
    def upsert_credit_quality(self, records: list[dict]) -> UpsertResult
    def upsert_category_returns(self, records: list[dict]) -> UpsertResult
```

Each method returns:
```python
@dataclass
class UpsertResult:
    inserted: int
    updated: int
    failed: int
    errors: list[dict]
```

Upsert strategy:
- Use SQLAlchemy `insert().on_conflict_do_update()` for PostgreSQL
- Conflict key for `fund_master`: `mstar_id`
- Conflict key for `nav_daily`: `(mstar_id, nav_date)`
- Conflict key for `risk_stats_monthly`: `(mstar_id, as_of_date)`
- Conflict key for `rank_monthly`: `(mstar_id, as_of_date)`
- Conflict key for `fund_holdings_snapshot`: `(mstar_id, portfolio_date)`
- Process in batches of 500 rows (configurable) — don't try to insert 750K NAV rows in one statement
- Wrap each batch in a transaction — one batch failure doesn't roll back others

### File 5: `backend/app/services/ingestion_service.py`

Orchestrates the full ingestion flow: parse → validate → upsert → audit.

```python
class IngestionService:
    """
    Orchestrates Morningstar feed ingestion.
    parse → validate → bulk upsert → log to ingestion_log + audit_trail.
    """

    def __init__(self, db: Session): ...

    def ingest_master_feed(self, csv_path: str) -> IngestionResult
    def ingest_nav_feed(self, csv_path: str) -> IngestionResult
    def ingest_risk_stats_feed(self, csv_path: str) -> IngestionResult
    def ingest_rank_feed(self, csv_path: str) -> IngestionResult
    def ingest_holdings_feed(self, csv_path: str) -> IngestionResult
    def ingest_category_returns_feed(self, csv_path: str) -> IngestionResult

    def ingest_all_pending(self, feed_dir: str) -> list[IngestionResult]
```

Each method:
1. Calls `FeedParser.parse_xxx_feed()` to get parsed records
2. Calls `IngestionRepository.upsert_xxx()` to save to DB
3. Creates `ingestion_log` entry (feed_name, records_processed, records_failed, duration_ms, status)
4. Creates `audit_trail` entry (actor="system/ingestion", action="DATA_INGESTED", details={feed_type, file, records, errors})
5. Returns `IngestionResult`:

```python
@dataclass
class IngestionResult:
    feed_type: str
    source_file: str
    status: str              # "SUCCESS", "PARTIAL", "FAILED"
    total_rows: int
    parsed_rows: int
    inserted: int
    updated: int
    failed: int
    duration_ms: int
    errors: list[dict]       # First 20 errors (don't log 10K errors)
```

`ingest_all_pending()` scans the feed directory for unprocessed CSVs, identifies the feed type by filename pattern (e.g. `master_*.csv`, `nav_*.csv`, `risk_*.csv`), and processes each one.

### File 6: `backend/app/services/marketpulse_client.py`

Thin HTTP client for MarketPulse bridge.

```python
class MarketPulseClient:
    """
    HTTP client for MarketPulse (localhost:8000).
    Read-only bridge — MF Pulse never writes to MarketPulse.
    All responses cached locally for configurable TTL.
    """

    def __init__(self, base_url: str, timeout: int = 30): ...

    def get_breadth_history(self, lookback: str = "1y") -> dict | None
    def get_sentiment(self) -> dict | None
    def get_sector_scores(self, period: str = "3M") -> list[dict] | None
    def get_market_picks(self) -> dict | None
    def health_check(self) -> bool
```

Rules:
- Use `httpx` (sync client, matching the rest of the codebase)
- Every method returns `None` on failure (not exceptions) — MarketPulse being down should never crash MF Pulse
- Cache responses in-memory with configurable TTL (default: 15 minutes for breadth/sentiment, 60 minutes for sectors)
- Log warnings on failure, never errors (MarketPulse is an optional enhancement, not a dependency)
- Include timeout handling — don't hang if MarketPulse is slow
