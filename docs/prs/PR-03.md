# PR-03: Six-Lens Classification Engine

## Objective
Build the core intelligence engine that classifies every mutual fund across six independent dimensions — Return, Risk, Consistency, Alpha, Efficiency, Resilience. Each lens produces a percentile score (0-100) within the fund's SEBI category and a human-readable tier label. No composite score. This is the analytical backbone of the entire platform.

---

## Context

PR-02 built the read APIs for raw fund data. PR-03 adds the **intelligence layer** — it reads from `nav_daily`, `risk_stats_monthly`, `rank_monthly`, and `fund_master`, computes six independent scores, and writes results to `fund_lens_scores` and `fund_classification`. Every computation is audited.

## Deliverables

### File 1: `backend/app/engines/lens_engine.py`

The pure computation engine. Takes data in, returns scores out. No DB access, no HTTP — fully testable in isolation.

```python
"""
Six-Lens Classification Engine

Each lens is a percentile rank (0-100) within the fund's SEBI category.
Higher = better for ALL lenses (Risk lens inverts so low-risk funds score high).
Then classified into 4 tiers per lens.

The engine receives pre-loaded data dicts. It never touches the database.
"""

from decimal import Decimal
from dataclasses import dataclass
from typing import Optional


# --- Tier Thresholds (configurable via engine_config) ---
TIER_THRESHOLDS = {
    "TIER_1": Decimal("75"),   # Top quartile
    "TIER_2": Decimal("50"),   # Above median
    "TIER_3": Decimal("25"),   # Below median
    # Below TIER_3 = TIER_4
}

# --- Tier Labels Per Lens ---
TIER_LABELS = {
    "return":      ["LEADER", "STRONG", "AVERAGE", "WEAK"],
    "risk":        ["LOW_RISK", "MODERATE", "ELEVATED", "HIGH_RISK"],
    "consistency": ["ROCK_SOLID", "CONSISTENT", "MIXED", "ERRATIC"],
    "alpha":       ["ALPHA_MACHINE", "POSITIVE", "NEUTRAL", "NEGATIVE"],
    "efficiency":  ["LEAN", "FAIR", "EXPENSIVE", "BLOATED"],
    "resilience":  ["FORTRESS", "STURDY", "FRAGILE", "VULNERABLE"],
}

ENGINE_VERSION = "1.0.0"


@dataclass
class LensResult:
    """Result for one fund across all six lenses."""
    mstar_id: str
    category_name: str
    # Raw percentile scores (0-100)
    return_score: Optional[Decimal]
    risk_score: Optional[Decimal]
    consistency_score: Optional[Decimal]
    alpha_score: Optional[Decimal]
    efficiency_score: Optional[Decimal]
    resilience_score: Optional[Decimal]
    # Tier classifications
    return_class: Optional[str]
    risk_class: Optional[str]
    consistency_class: Optional[str]
    alpha_class: Optional[str]
    efficiency_class: Optional[str]
    resilience_class: Optional[str]
    # Headline tag
    headline_tag: str
    # Metadata
    data_completeness_pct: Decimal
    available_horizons: int
    engine_version: str = ENGINE_VERSION


class LensEngine:
    """
    Computes six-lens classification for all funds within a single SEBI category.
    Call compute_category() once per category with all funds' data pre-loaded.
    """

    def compute_category(
        self,
        category_name: str,
        fund_ids: list[str],
        latest_returns: dict[str, dict],      # mstar_id → {return_1y, return_3y, return_5y, ...}
        risk_stats: dict[str, dict],           # mstar_id → {sharpe_3y, std_dev_3y, max_drawdown_3y, ...}
        ranks: dict[str, dict],                # mstar_id → {quartile_1y, quartile_3y, ...}
        fund_master: dict[str, dict],          # mstar_id → {net_expense_ratio, turnover_ratio, ...}
        calendar_year_returns: dict[str, dict], # mstar_id → {calendar_year_return_1y, ..., _10y}
    ) -> list[LensResult]:
        """
        Compute all six lenses for every fund in a category.
        Returns one LensResult per fund.
        """

    # --- Individual Lens Computations ---

    def _compute_return_lens(
        self,
        fund_ids: list[str],
        latest_returns: dict[str, dict],
    ) -> dict[str, Optional[Decimal]]:
        """
        RETURN LENS: Does it make money?

        Inputs: Return_1Y (weight 20%), Return_3Y (weight 35%), Return_5Y (weight 45%)
        Method:
          1. For each fund, compute weighted return: 0.20*R1Y + 0.35*R3Y + 0.45*R5Y
          2. If a horizon is missing (fund too young), redistribute weight proportionally
          3. Rank all funds in category by weighted return
          4. Convert rank to percentile (0-100)
        """

    def _compute_risk_lens(
        self,
        fund_ids: list[str],
        risk_stats: dict[str, dict],
    ) -> dict[str, Optional[Decimal]]:
        """
        RISK LENS: How bumpy is the ride?

        Inputs: StdDev_3Y, MaxDrawdown_3Y, Beta_3Y, Downside_Capture_3Y (equal weight)
        Method:
          1. For each metric, rank within category (INVERTED — lower risk = higher rank)
          2. Average the 4 percentile ranks
          3. Result: 0-100 where 100 = lowest risk in category
        Note: All metrics are "lower is better" so we invert the ranking.
        """

    def _compute_consistency_lens(
        self,
        fund_ids: list[str],
        ranks: dict[str, dict],
        risk_stats: dict[str, dict],
    ) -> dict[str, Optional[Decimal]]:
        """
        CONSISTENCY LENS: Can you count on it?

        Inputs:
          - Quartile rank history: how often in Q1 or Q2 across available periods
            (quartile_1y, quartile_3y, quartile_5y — count periods where quartile <= 2)
          - Calendar year percentile ranks: how often in top half
            (cal_year_pctile_1y through _10y — count where pctile <= 50)
          - Sortino_3Y: rewards consistency of positive returns over raw magnitude

        Method:
          1. Quartile consistency score = (periods in Q1 or Q2) / (total periods with data) * 100
          2. Calendar year consistency = (years in top half) / (total years with data) * 100
          3. Sortino percentile within category
          4. Final = 40% quartile + 30% calendar + 30% sortino
        """

    def _compute_alpha_lens(
        self,
        fund_ids: list[str],
        risk_stats: dict[str, dict],
        latest_returns: dict[str, dict],
        category_avg_returns: dict,
    ) -> dict[str, Optional[Decimal]]:
        """
        ALPHA LENS: Is the manager adding value?

        Inputs:
          - Alpha_3Y, Alpha_5Y (from risk_stats — benchmark-relative alpha)
          - Information_Ratio_3Y, Information_Ratio_5Y
          - Excess return vs category average: fund_return - category_avg (3Y, 5Y)

        Method:
          1. Percentile rank each metric within category (higher = better)
          2. Weight: 5Y metrics × 60%, 3Y metrics × 40%
          3. Average weighted percentile ranks
        """

    def _compute_efficiency_lens(
        self,
        fund_ids: list[str],
        latest_returns: dict[str, dict],
        fund_master: dict[str, dict],
    ) -> dict[str, Optional[Decimal]]:
        """
        EFFICIENCY LENS: Is it worth the cost?

        Inputs:
          - Net expense ratio (lower = better)
          - Turnover ratio (lower = better — less churn)
          - Return per unit expense: Return_3Y / expense_ratio (higher = better)

        Method:
          1. Expense percentile (inverted — lowest expense = 100)
          2. Turnover percentile (inverted — lowest turnover = 100)
          3. Return-per-expense percentile (higher = 100)
          4. Final = 40% expense + 20% turnover + 40% return-per-expense
        """

    def _compute_resilience_lens(
        self,
        fund_ids: list[str],
        risk_stats: dict[str, dict],
        calendar_year_returns: dict[str, dict],
    ) -> dict[str, Optional[Decimal]]:
        """
        RESILIENCE LENS: How does it behave in bad markets?

        Inputs:
          - MaxDrawdown_3Y (inverted — smaller drawdown = better)
          - Downside_Capture_3Y (inverted — lower capture = better)
          - Upside/Downside capture ratio: capture_up_3y / capture_down_3y (higher = better)
          - Worst calendar year return: min(calendar_year_return_1y through _10y)
            Percentile rank of worst year (less negative = better)

        Method:
          1. Percentile rank each metric within category
          2. Equal weight across available metrics
          3. Funds with no drawdown data get None (not scored)
        """

    # --- Utility Methods ---

    def _percentile_rank(
        self,
        values: dict[str, Optional[Decimal]],
        higher_is_better: bool = True,
    ) -> dict[str, Optional[Decimal]]:
        """
        Convert raw values to percentile ranks (0-100) within the group.
        None values get None rank. Uses competition ranking.
        """

    def _classify_tier(
        self,
        score: Optional[Decimal],
        lens_name: str,
    ) -> Optional[str]:
        """Map a 0-100 percentile score to a tier label."""
        if score is None:
            return None
        labels = TIER_LABELS[lens_name]
        if score >= TIER_THRESHOLDS["TIER_1"]:
            return labels[0]
        if score >= TIER_THRESHOLDS["TIER_2"]:
            return labels[1]
        if score >= TIER_THRESHOLDS["TIER_3"]:
            return labels[2]
        return labels[3]

    def _generate_headline(self, result: LensResult) -> str:
        """
        Generate a human-readable headline from the top lens classifications.

        Logic:
          1. Collect all non-None classifications
          2. Pick the 2-3 most distinctive (furthest from average)
          3. Combine into natural language

        Examples:
          - "Consistent alpha generator with low risk"
          - "Strong returns but elevated risk and expensive fees"
          - "Resilient performer with moderate returns"
          - "Efficient fund with mixed consistency"
        """

    def _compute_data_completeness(
        self,
        mstar_id: str,
        latest_returns: dict,
        risk_stats: dict,
        ranks: dict,
    ) -> tuple[Decimal, int]:
        """
        Compute what % of expected data points are available.
        Returns (completeness_pct, available_horizons).
        A fund with only 1Y data scores lower completeness than one with 1Y/3Y/5Y.
        """
```

Orchestrates the lens computation: loads data from DB, calls the engine, saves results, writes audit trail.

```python
class LensService:
    """
    Orchestrates six-lens computation across all categories.
    Loads data → calls LensEngine → saves to fund_lens_scores + fund_classification → audits.
    """

    def __init__(self, db: Session): ...

    def compute_all_categories(self) -> dict:
        """
        Run lens computation for every SEBI category.
        Returns summary: { categories_processed, funds_scored, duration_ms, errors }
        """

    def compute_single_category(self, category_name: str) -> dict:
        """
        Run lens computation for one category.
        Returns: { category, funds_scored, results_summary }
        """

    def compute_single_fund(self, mstar_id: str) -> LensResult | None:
        """
        Compute lenses for a single fund (uses its category peers for ranking).
        Useful for on-demand recomputation after FM override.
        """

    def _load_category_data(self, category_name: str) -> dict:
        """
        Load all data needed for lens computation for one category:
        - fund_ids: list of eligible mstar_ids
        - latest_returns: from nav_daily (latest row per fund)
        - risk_stats: from risk_stats_monthly (latest row per fund)
        - ranks: from rank_monthly (latest row per fund)
        - fund_master: expense ratio, turnover, etc.
        - calendar_year_returns: from nav_daily
        - category_avg_returns: from category_returns_daily
        All loaded in bulk (one query per table, not per fund).
        """

    def _save_results(self, results: list[LensResult], computed_date: date) -> None:
        """
        Save lens scores and classifications to DB.
        Uses upsert (ON CONFLICT mstar_id, computed_date).
        Creates audit_trail entry.
        """
```

## Key Design Decisions

1. **Percentile within category, not absolute score.** A fund scoring 80 in "Large Cap" means it's in the top 20% of large cap funds — not that it has some absolute quality of 80. This makes scores comparable within a category but NOT across categories (by design).

2. **Engine is pure computation.** LensEngine has zero DB dependencies. You can unit test it with fake data. The service layer handles DB I/O. This separation is critical.

3. **Missing data → None, not zero.** A fund without 5Y track record gets None for 5Y-dependent metrics, not zero. The weight is redistributed to available horizons. This prevents young funds from being unfairly penalized.

4. **All six lenses are independent.** A fund's Return score does not influence its Risk score. They are computed from different metrics. The only shared input is the peer group (category).

5. **Headline tag is generative, not template-based.** It reads the actual classifications and constructs a natural sentence. "Consistent alpha generator with fortress-level resilience" is more useful than "Return: LEADER, Risk: LOW_RISK, ...".
