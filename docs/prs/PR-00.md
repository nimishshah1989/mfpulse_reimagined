# PR-0: Project Foundation ‚Äî Core App + Database Schema + Health Endpoint

## Objective
Stand up a deployable FastAPI application with the complete database schema, health endpoint, and Nginx configuration. At the end of this PR, `mfpulse.jslwealth.in/health` returns a 200 OK with database connectivity confirmed.

---

## Pre-requisites (manual ops, done by Nimish before this PR)

### 1. Create the `mf_pulse` database
SSH into the server and create the database in the existing PostgreSQL container:
```bash
ssh -i ~/.ssh/jsl-wealth-key.pem ubuntu@13.206.34.214
sudo docker exec -it fie2-db-1 psql -U fie -c "CREATE DATABASE mf_pulse;"
```

### 2. Create Nginx config
```bash
sudo tee /etc/nginx/sites-available/mfpulse.conf << 'NGINX'
server {
    listen 80;
    server_name mfpulse.jslwealth.in;

    location / {
        proxy_pass http://127.0.0.1:8001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 120s;
    }
}
NGINX

sudo ln -sf /etc/nginx/sites-available/mfpulse.conf /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

### 3. DNS
Point `mfpulse.jslwealth.in` A record to `13.206.34.214` (same as other JSL subdomains).

## Deliverables

### File 1: `backend/app/core/config.py`
Pydantic Settings class loading from environment variables. All fields documented in `.env.example`. Singleton via `@lru_cache`. Validates critical config on load.

### File 2: `backend/app/core/database.py`
SQLAlchemy 2.0 sync engine. `DeclarativeBase` subclass. Session factory. `get_db()` FastAPI dependency. `init_db()` for dev.

Connect to the existing PostgreSQL container: `postgresql://fie:{password}@localhost:5432/mf_pulse`
(In Docker, the container uses the Docker network. In production, MF Pulse container connects via `host.docker.internal:5432` or the Docker bridge network.)

### File 3: `backend/app/core/exceptions.py`
Custom exception hierarchy:
- `MFPulseError(message, code, details)` ‚Äî base
- `NotFoundError`, `ValidationError`, `IngestionError`, `MorningstarError`, `MarketPulseError`, `EngineError`

### File 4: `backend/app/main.py`
FastAPI application with:
- CORS middleware (origins from config)
- Global exception handler for `MFPulseError` ‚Üí JSON error response
- Mount `/api/v1` router
- Health endpoint at `/health` (not under /api/v1)
- Static frontend serving from `web/out/` (matching MarketPulse pattern)
- Startup log with version, environment, DB status

### File 5: `backend/app/api/v1/router.py`
Central router that mounts sub-routers. For PR-0, only `system.py` is mounted.

### File 6: `backend/app/api/v1/system.py`
Endpoints:
- `GET /api/v1/system/health` ‚Äî DB connectivity check, returns uptime, version, db status, timestamp
- `GET /api/v1/system/config` ‚Äî Non-sensitive config: app version, env, scheduler status, feed schedule

## Acceptance Criteria

1. `docker compose up` starts the container and connects to PostgreSQL
2. `GET /health` returns `{"status": "ok", "database": "connected", "version": "0.1.0"}`
3. `GET /api/v1/system/health` returns detailed health with DB stats
4. `alembic upgrade head` creates all tables without errors
5. All tables exist in `mf_pulse` database with correct columns, types, indexes, and constraints
6. No hardcoded credentials anywhere ‚Äî all from environment
7. `pytest` passes (at minimum: health endpoint test, config loading test)
