# PR-13: Production Hardening — Auth, CI/CD, Frontend Efficiency, Monitoring

## Objective
Make MF Pulse production-solid. Three pillars: (1) Security and ops — API auth, CI/CD pipeline, health checks, access code rotation. (2) Frontend efficiency — caching layer so 13,365 funds aren't re-fetched on every page load. (3) Monitoring — structured logging, system status, error standardization. After this PR, the platform is production-ready with sub-second page loads and zero redundant data fetching.

---

## PILLAR 1: FRONTEND EFFICIENCY — NO REDUNDANT DATA PULLS

This is the most important section. Currently `fetchAllFunds()` loads ALL 13,365 funds (paginating 500/page = 27 API calls = ~7MB) every time someone opens the Universe Explorer. Every tab switch, every browser refresh = full reload. This must stop.

### 1A. Server-Side: API Response Caching + Compression

#### File: Update `backend/app/main.py`

**Gzip compression middleware:**
```python
from fastapi.middleware.gzip import GZipMiddleware
app.add_middleware(GZipMiddleware, minimum_size=1000)  # Compress responses > 1KB
```
This alone cuts the 7MB payload to ~1-1.5MB on the wire.

**Cache-Control headers on read endpoints:**
```python
@app.middleware("http")
async def add_cache_headers(request: Request, call_next):
    response = await call_next(request)
    if request.method == "GET" and request.url.path.startswith("/api/v1/"):
        path = request.url.path
        if "/funds" in path or "/categories" in path or "/amcs" in path:
            response.headers["Cache-Control"] = "public, max-age=300"  # 5 min
        elif "/lens/scores" in path:
            response.headers["Cache-Control"] = "public, max-age=600"  # 10 min
        elif "/marketpulse" in path:
            response.headers["Cache-Control"] = "public, max-age=60"   # 1 min
        elif "/simulation" in path:
            response.headers["Cache-Control"] = "no-cache"  # Always fresh
    return response
```

**Bulk endpoint for Universe Explorer:**

Instead of paginating 27 times, create ONE endpoint optimized for the scatter chart:

```python
@router.get("/universe")
def get_universe_data(db: Session = Depends(get_db)) -> dict:
    """Returns ALL funds with lens scores in a single optimized response.
    
    Only includes fields the Universe Explorer needs:
    mstar_id, fund_name, amc_name, category_name, broad_category,
    return_score, risk_score, consistency_score, alpha_score,
    efficiency_score, resilience_score, return_class, risk_class,
    return_1y, return_3y, aum.
    
    Returns compact JSON — no nulls, no extra fields.
    Response is ~1.5MB raw, ~300KB gzipped.
    Cached server-side for 10 minutes.
    """
```

This endpoint does a single JOIN query (fund_master + lens_scores) and returns only the 15 fields the scatter chart needs — not the full 40+ column fund record. Response drops from ~7MB to ~1.5MB raw, ~300KB gzipped.

### 1B. Client-Side: Data Cache Layer

#### File: `web/src/lib/cache.js`

Module-level cache that persists across tab switches (page navigations within the SPA). Data is fetched once and reused until explicitly invalidated or TTL expires.

```javascript
/**
 * Client-side data cache with TTL.
 * 
 * Persists across tab switches (module-level, not component state).
 * Automatically invalidates after TTL.
 * Shared across all pages — Universe, Fund360, Sectors all read from same cache.
 *
 * Usage:
 *   const funds = await cachedFetch('universe', fetchUniverseData, 600);
 *   // Second call within 600s returns instantly from cache
 */

const _cache = {};

export async function cachedFetch(key, fetchFn, ttlSeconds = 300) {
  const entry = _cache[key];
  if (entry && Date.now() - entry.timestamp < ttlSeconds * 1000) {
    return entry.data;
  }
  const data = await fetchFn();
  _cache[key] = { data, timestamp: Date.now() };
  return data;
}

export function invalidateCache(key) {
  if (key) delete _cache[key];
  else Object.keys(_cache).forEach(k => delete _cache[k]);
}

export function getCacheStatus() {
  return Object.entries(_cache).map(([key, entry]) => ({
    key,
    age_seconds: Math.round((Date.now() - entry.timestamp) / 1000),
    size: JSON.stringify(entry.data).length,
  }));
}
```

Now: first visit loads data in ~1 second. Tab switch away and back to Universe = instant (from cache). Browser refresh = fresh fetch. Cache invalidates after 10 minutes.

### 1C. Page-Level Data Loading Strategy

Each page should only fetch what it needs, when it needs it:

| Page | Data Loaded | Cache TTL | Size (gzipped) |
|------|-------------|-----------|----------------|
| Universe Explorer | `/funds/universe` (bulk compact) | 10 min | ~300KB |
| Fund 360 | `/funds/{id}` + `/lens/scores/{id}` (single fund) | 5 min | ~5KB |
| Sector Intelligence | `/marketpulse/sectors` + `/marketpulse/regime` | 1 min | ~10KB |
| Simulation Lab | `/funds/{id}/nav?period=max` (on demand, per fund) | 5 min | ~50KB |
| Strategy Builder | `/strategies` (list, on demand) | No cache | ~5KB |
| Pulse Dashboard | `/marketpulse/*` + `/lens/scores?limit=20` + `/system/status` | 1 min | ~20KB |

**Rule: No page should fetch more than 500KB on load.** The Universe Explorer was violating this at 7MB. The bulk endpoint + gzip fixes it.

## PILLAR 3: SECURITY & MONITORING

### 3A. API Authentication

#### File: `backend/app/core/auth.py`

```python
"""API key auth for write endpoints. Read endpoints stay public."""

def require_admin_key(request: Request):
    settings = get_settings()
    if not settings.admin_api_key:
        return  # No key = auth disabled (dev mode)
    key = request.headers.get("X-API-Key")
    if key != settings.admin_api_key:
        raise HTTPException(status_code=401, detail="Invalid or missing API key")
```

**Protected:** All POST/PUT/DELETE on ingestion, lens compute, strategies, overrides, job triggers.

**Public:** All GET endpoints.

### 3B. Rate Limiting

#### File: `backend/app/core/rate_limit.py`

### 3D. Structured Request Logging

Middleware that logs every request with: request_id, method, path, status code, duration_ms, client IP. Returns X-Request-ID header. Never exposes stack traces to clients.

### 3E. Access Code Auto-Rotation

Weekly scheduler job: if access code expires within 14 days, automatically create a new 90-day code using the account management API. Log result to audit trail.

### 3F. Error Response Standardization

Every error returns:
```json
{"success": false, "data": null, "error": {"code": "ERROR_CODE", "message": "...", "details": {}}}
```

Codes: `FUND_NOT_FOUND`, `VALIDATION_ERROR`, `MORNINGSTAR_ERROR`, `UNAUTHORIZED`, `RATE_LIMITED`, `INTERNAL_ERROR`
