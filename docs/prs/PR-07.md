# PR-07: Frontend Scaffold + Universe Explorer

## Objective
Stand up the Next.js frontend and build the flagship screen — the Universe Explorer with three visualization modes (bubble scatter, treemap, heatmap). After this PR, `mfpulse.jslwealth.in` serves a working fund exploration interface connected to the live backend API.

---

## Context

This is the first frontend PR. The backend is complete (PRs 00–06): 40+ API endpoints, 3 computation engines (lens, signal, simulation), 22 database tables, scheduled jobs. The frontend is a Next.js 14 static export served by FastAPI (same Docker container, same pattern as MarketPulse).

**Design system**: Jhaveri Intelligence Platform light theme — white backgrounds, teal-600 primary, Inter font, Indian number formatting (₹, Cr, L), font-mono for all financials. See `ui-design-system` skill for the full spec.

**Key APIs this PR consumes**:
- `GET /api/v1/funds?category=&amc=&search=&sort_by=&limit=&offset=` — fund list with latest NAV + returns
- `GET /api/v1/lens/scores?category=&limit=` — all lens scores (return, risk, consistency, alpha, efficiency, resilience)
- `GET /api/v1/funds/categories` — all SEBI categories with fund counts
- `GET /api/v1/funds/amcs` — all AMCs with fund counts
- `GET /api/v1/funds/{mstar_id}` — fund deep-dive (used in detail panel)
- `GET /api/v1/marketpulse/regime` — market regime for header ticker

### File 4: `web/src/lib/api.js`

Thin API client. Every call returns `{ success, data, meta, error }` envelope. Handles errors uniformly.

```js
const BASE = process.env.NEXT_PUBLIC_API_URL || '';

async function apiFetch(path, options = {}) {
  const res = await fetch(`${BASE}${path}`, {
    headers: { 'Content-Type': 'application/json', ...options.headers },
    ...options,
  });
  if (!res.ok) {
    const err = await res.json().catch(() => ({ error: res.statusText }));
    throw new Error(err.error || `API error ${res.status}`);
  }
  return res.json();
}

// Fund APIs
export const fetchFunds = (params) => apiFetch(`/api/v1/funds?${new URLSearchParams(params)}`);
```

### File 5: `web/src/lib/format.js`

Indian number formatting utilities.

```js
// ₹1,23,45,678 format
export function formatINR(num, decimals = 2) {
  if (num == null) return '—';
  const n = Number(num);
  const sign = n < 0 ? '−' : '';
  const abs = Math.abs(n);
  // Indian grouping: last 3, then groups of 2
  const [intPart, decPart] = abs.toFixed(decimals).split('.');
  const lastThree = intPart.slice(-3);
  const rest = intPart.slice(0, -3);
  const formatted = rest.replace(/\B(?=(\d{2})+(?!\d))/g, ',') + (rest ? ',' : '') + lastThree;
  return `${sign}₹${formatted}${decPart ? '.' + decPart : ''}`;
}

// Smart shorthand: ₹12.5Cr, ₹48.3L
export function formatAUM(num) {
  if (num == null) return '—';
  const n = Number(num);
  const abs = Math.abs(n);
  const sign = n < 0 ? '−' : '';
  if (abs >= 100) return `${sign}₹${(abs / 100).toFixed(abs >= 1000 ? 0 : 1)}Cr`;
  if (abs >= 1) return `${sign}₹${abs.toFixed(2)}Cr`;
  return `${sign}₹${(abs * 100).toFixed(1)}L`;
}

// +18.4% or −3.2%
export function formatPct(num, decimals = 1) {
  if (num == null) return '—';
  const n = Number(num);
  return `${n >= 0 ? '+' : '−'}${Math.abs(n).toFixed(decimals)}%`;
}

// Score display: 84/100
export function formatScore(num) {
  if (num == null) return '—';
  return `${Math.round(Number(num))}`;
}
```

### File 8: `web/src/pages/universe.jsx` — THE FLAGSHIP PAGE

Three visualization modes in tabs: **Scatter** (default) | **Treemap** | **Heatmap**

#### Data Loading

On mount, fetch:
1. `fetchLensScores({ limit: 5000 })` — all funds with 6 lens scores
2. `fetchCategories()` — for filter panel category checkboxes
3. `fetchAMCs()` — for filter panel AMC multi-select

Combine into a single data array where each fund has: `mstar_id`, `fund_name`, `amc_name`, `category_name`, `broad_category`, `return_score`, `risk_score`, `consistency_score`, `alpha_score`, `efficiency_score`, `resilience_score`, `return_class` (tier), `aum`, `return_1y`, `return_3y`.

#### Filter Panel (left sidebar, always visible, 280px wide)

1. **Axis Selectors** (scatter mode only): Two dropdowns for X-axis and Y-axis. Options: Return, Risk, Consistency, Alpha, Efficiency, Resilience. Default: X=Return, Y=Risk.

2. **Category Checkboxes**: All SEBI categories grouped by broad_category (Equity, Debt, Hybrid, Other). Each checkbox toggles funds in/out. "Select All" / "Clear" buttons.

3. **AMC Multi-Select**: Searchable dropdown, select multiple AMCs. Default: all selected.

4. **Lens Score Sliders**: 6 range sliders (0–100), one per lens. Default: all at 0 (show everything). Dragging a slider to 60 hides all funds scoring below 60 on that lens. Label shows current threshold + count of funds passing.

5. **AUM Range**: Dual-handle slider for min/max AUM.

6. **Quick Presets**: Buttons that set filter combinations:
   - "Elite Funds" → all 6 lens sliders to 70+
   - "Large Cap Quality" → category=Large Cap, Return≥60, Consistency≥60
   - "High Alpha Small" → category=Small Cap, Alpha≥75
   - "Defensive Picks" → Risk≥70, Resilience≥70
   - "Reset All" → clear everything

7. **Active filter count** shown in header: "Showing 847 of 3,124 funds"

#### Mode 1: Bubble Scatter (D3.js)

**Canvas**: Full remaining width (100% - 280px filter panel), height 600px. White background.

**Axes**: X and Y determined by axis selector dropdowns. Both 0–100 scale. Axis labels: lens name + "(score 0–100)". Grid lines at 20, 40, 60, 80 in slate-100.

**Bubbles**: One circle per fund. Radius = `Math.sqrt(aum) * scaleFactor` clamped between 3px and 25px (log scale so small funds visible). Fill color by broad_category:
- Equity: teal-500 at 60% opacity
- Debt: emerald-500 at 60% opacity
- Hybrid: amber-500 at 60% opacity
- Other: slate-400 at 60% opacity

Stroke: same color at 100% opacity, 1px.

**Quadrant overlay**: Faint dashed lines at x=50 and y=50 creating 4 quadrants. Top-right labeled "Sweet spot" in slate-300 italic. Bottom-left labeled "Below average" in slate-300 italic.

**Interactions**:
- **Hover**: Tooltip appears showing fund_name, AMC, category, all 6 lens scores as horizontal bars (0–100 with fill), 1Y return, 3Y return. Tooltip follows cursor, positioned to avoid viewport overflow.
- **Click**: Right-side detail panel (320px) slides in from right edge. Shows full fund card: name, AMC, category, all 6 lens scores with tier badges, returns table, and two action buttons: "Deep Dive →" (navigates to fund360 tab with this fund), "Simulate →" (navigates to simulation tab with this fund). Panel has close X button.
- **Zoom**: D3 zoom behavior. Scroll to zoom, drag to pan. Reset button in top-right corner.
- **Filter animation**: When filters change, bubbles animate to new positions / appear / disappear with 400ms ease-out transition. Filtered-out bubbles shrink to 0 and fade out simultaneously. New bubbles grow from 0.

**Legend**: Bottom of chart area. Color swatches for each broad_category with fund count in parentheses.

**Performance**: For 3,000+ points, use D3 canvas rendering (not SVG). Draw to `<canvas>` element. Hit detection via quadtree for hover/click. This is critical — SVG with 3,000 circles will lag on filter animations.

#### Mode 2: Treemap (D3.js)

**Layout**: Full width, height 500px. D3 treemap layout with squarified tiling.

**Hierarchy**: Top level = broad_category (Equity, Debt, Hybrid). Second level = SEBI category (Large Cap, Mid Cap, etc.). Leaf = individual fund.

**Sizing**: Each fund's rectangle area proportional to AUM. If AUM is null, use equal sizing within category.

**Coloring**: User selects which lens score colors the treemap from a dropdown (default: Return). Color scale: sequential from slate-100 (score 0) through teal-100 (score 40) to teal-600 (score 80) to emerald-600 (score 100). Funds with no lens score = slate-200.

**Labels**: Category names shown in bold at the top of each category section. Individual fund names shown only if the rectangle is large enough (> 80px width AND > 40px height). Truncated with ellipsis otherwise.

**Interactions**:
- **Hover**: Tooltip with fund name, category, AUM, selected lens score.
- **Click**: Same right-side detail panel as scatter mode.
- **Breadcrumb**: Click a category section → zoom into that category showing only its funds, with a "← Back to all" breadcrumb.

#### Mode 3: Heatmap (SVG/HTML)

**Layout**: HTML table/grid. Rows = SEBI categories (sorted by broad_category group). Columns = lens score quintile buckets (0–20, 20–40, 40–60, 60–80, 80–100).

**User selects which lens** to analyze from a dropdown (default: Return).

**Cells**: Background color intensity proportional to fund count in that bucket. Scale: white (0 funds) → teal-50 (few) → teal-200 (moderate) → teal-500 (many) → teal-800 (most). Cell text: fund count number in center, bold.

**Row totals**: Rightmost column shows total funds and average score for that category. Average score colored: ≥70 emerald, ≥50 teal, <50 amber.

**Column totals**: Bottom row shows total funds per quintile across all categories.

**Interactions**:
- **Hover cell**: Tooltip showing "Large Cap: 18 funds scoring 60–80 on Return. Top 3: [fund names with scores]."
- **Click cell**: Filter the scatter view to just those funds (switch to scatter tab, set category + lens range).

## Key Design Decisions

1. **D3 canvas for scatter, not SVG.** 3,000 bubbles in SVG will lag during filter animations. Canvas with D3 quadtree gives 60fps hover detection and smooth transitions.

2. **Lens scores are the primary data, not raw returns.** The scatter plots lens scores (0–100 normalized), not raw percentages. This makes cross-category comparison meaningful — a "90 on Return" means top decile within its category, whether it's a Large Cap or Small Cap fund.

3. **Any lens pair on axes.** Not locked to risk/return. Alpha vs Consistency reveals "skilled and reliable" funds. Efficiency vs Resilience reveals "low-cost and crash-resistant" funds. Letting the user choose drives discovery.

4. **Treemap sized by AUM, colored by quality.** This instantly shows "where is money concentrated vs where is quality concentrated?" If Large Cap has 60% of AUM but only 30% of funds score above 70, that's a visible insight.

5. **Heatmap → scatter drill-through.** Clicking a heatmap cell switches to scatter mode with those exact filters. The heatmap is the analytical summary, the scatter is the exploration. They're connected, not isolated.

6. **Filter panel always visible.** No toggle, no collapse. This is a power-user tool — the filters ARE the product. Hide them behind a button and you've killed the interaction model.

7. **Static export, not SSR.** Same deployment pattern as MarketPulse. FastAPI serves the pre-built HTML/JS/CSS. API calls happen client-side. No Node.js server in production.
