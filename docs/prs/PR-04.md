# PR-04: Simulation Engine — Signal Framework + SIP/Lumpsum Simulator

## Objective
Build the simulation engine that backtests investment strategies across real NAV history. Supports 4 investment modes (Pure SIP, SIP + Signal Top-ups, Pure Lumpsum, Hybrid), a configurable multi-source signal framework with AND/OR conditions, and produces comprehensive output statistics (XIRR, max drawdown, Sharpe, rolling returns, signal hit rate).

---

## Context

This is the biggest single PR. It combines:
1. The MF Simulator's proven SIP + breadth top-up concept (from `mf-simulator-main/`)
2. MarketPulse's momentum/signal intelligence (from `fie2-main/services/compass_signals.py`)
3. Enhanced signal framework with AND/OR combinatorial logic (new)
4. Four investment modes instead of one (new)
5. Enhanced output statistics (new)

The engine is pure computation — takes NAV arrays and signal data in, returns simulation results out. No DB access inside the engine.

### File 1: `backend/app/engines/signal_engine.py`

The configurable signal framework. Evaluates market conditions against user-defined rules to determine when to deploy extra capital.

```python
"""
Signal Engine — Multi-source signal evaluation with AND/OR combinatorial logic.

Signal sources:
  - MarketPulse breadth (stocks above 21/50/200 EMA, new lows, advance-decline)
  - MarketPulse sentiment (composite score, 0-100)
  - MarketPulse sector RS (count of leading sectors)
  - Index momentum (NIFTY 50 vs 200 SMA — above or below)
  - VIX level (from index data)

Each SignalRule defines:
  - A set of conditions (signal_name, operator, threshold)
  - Logic: AND (all must be true) or OR (any must be true)
  - Multiplier: how much extra to deploy (1x = same as SIP, 3x = triple)
  - Cooloff: minimum days between triggers of this rule
"""

from dataclasses import dataclass, field
from datetime import date
from decimal import Decimal
from typing import Optional


@dataclass(frozen=True)
class SignalCondition:
    """One condition to evaluate."""
    signal_name: str       # Key into the signal_data dict
    operator: str          # "BELOW", "ABOVE", "CROSSES_BELOW", "CROSSES_ABOVE"
    threshold: float       # Threshold value

    def evaluate(self, current_value: float, previous_value: float | None = None) -> bool:
        """Evaluate this condition against current (and optional previous) value."""


@dataclass(frozen=True)
class SignalRule:
    """A named rule with conditions, logic, and deployment parameters."""
    name: str
    conditions: list[SignalCondition]
    logic: str = "AND"               # "AND" or "OR"
    multiplier: float = 1.0          # Top-up = SIP amount × multiplier
    cooloff_days: int = 30           # Min days between triggers

    def evaluate(self, signal_data: dict, previous_signal_data: dict | None = None) -> bool:
        """
        Evaluate all conditions against current signal data.
        AND: all conditions must be True.
        OR: at least one condition must be True.
        Returns True if the rule fires.
        """


@dataclass
class SignalEvent:
    """A fired signal event."""
    date: date
    rule_name: str
    multiplier: float
    conditions_met: list[str]   # Which conditions were True
    signal_snapshot: dict       # Signal values at the time


# --- Pre-built Rule Templates ---

DEFAULT_RULES: list[SignalRule] = [
    SignalRule(
        name="Mild correction",
        conditions=[
            SignalCondition("breadth_pct_above_21ema", "BELOW", 30.0),
        ],
        logic="AND",
        multiplier=1.0,
        cooloff_days=30,
    ),
    SignalRule(
        name="Moderate panic",
        conditions=[
            SignalCondition("breadth_pct_above_200ema", "BELOW", 40.0),
            SignalCondition("sentiment_composite", "BELOW", 35.0),
        ],
        logic="AND",
        multiplier=2.0,
        cooloff_days=21,
    ),
    SignalRule(
        name="Deep panic — max deploy",
        conditions=[
            SignalCondition("breadth_pct_above_200ema", "BELOW", 25.0),
            SignalCondition("sentiment_composite", "BELOW", 20.0),
        ],
        logic="AND",
        multiplier=3.0,
        cooloff_days=14,
    ),
    SignalRule(
        name="Trend recovery",
        conditions=[
            SignalCondition("nifty_above_200sma", "ABOVE", 0),
            SignalCondition("breadth_pct_above_50ema", "ABOVE", 60.0),
        ],
        logic="AND",
        multiplier=1.5,
        cooloff_days=45,
    ),
]


class SignalEngine:
    """
    Evaluates signal rules against daily signal data.
    Returns list of SignalEvents for a date range.
    """

    def __init__(self, rules: list[SignalRule] | None = None):
        self.rules = rules or DEFAULT_RULES

    def evaluate_range(
        self,
        signal_data_series: dict[date, dict],
        start_date: date,
        end_date: date,
    ) -> list[SignalEvent]:
        """
        Walk through each date in signal_data_series.
        For each date, evaluate all rules. If any fires (respecting cooloff),
        create a SignalEvent. If multiple rules fire on the same day,
        the one with the highest multiplier wins.
        Returns chronologically sorted list of SignalEvents.
        """

    def evaluate_single_day(
        self,
        signal_data: dict,
        previous_signal_data: dict | None,
        active_cooloffs: dict[str, date],  # rule_name → last_fired_date
        current_date: date,
    ) -> SignalEvent | None:
        """Evaluate all rules for one day. Returns highest-multiplier event or None."""
```

### File 2: `backend/app/engines/simulation_engine.py`

The core simulator. Takes NAV time series + signal events + parameters, produces a full simulation result.

```python
"""
SIP/Lumpsum Simulation Engine

4 investment modes:
  Mode 1: Pure SIP — fixed amount on fixed date (5th of month), no signals
  Mode 2: SIP + Signal Top-ups — regular SIP + extra on signal events
  Mode 3: Pure Lumpsum — deploy full amount at start date
  Mode 4: Hybrid — SIP + lumpsum reserve deployed in chunks on deep signals

All computations use Decimal. NAV values from Morningstar (already in DB).
Signal events from SignalEngine.

The engine takes numpy/pandas arrays in, returns structured results out.
No DB dependency.
"""

from dataclasses import dataclass, field
from datetime import date
from decimal import Decimal
from typing import Optional

from app.engines.signal_engine import SignalEvent


@dataclass(frozen=True)
class SimulationParams:
    """All parameters for one simulation run."""
    # Investment mode
    mode: str = "SIP"                        # SIP, SIP_SIGNAL, LUMPSUM, HYBRID

    # SIP parameters (for SIP, SIP_SIGNAL, HYBRID modes)
    sip_amount: Decimal = Decimal("10000")   # Monthly SIP amount (₹)
    sip_day: int = 5                         # Day of month for SIP

    # Lumpsum parameters (for LUMPSUM, HYBRID modes)
    lumpsum_amount: Decimal = Decimal("0")   # Total lumpsum to deploy
    lumpsum_deploy_pct: Decimal = Decimal("100")  # % of reserve per signal (HYBRID)

    # Signal top-up parameters (for SIP_SIGNAL, HYBRID modes)
    # Signal rules are passed separately to the engine

    # Time range
    start_date: date = date(2020, 1, 1)
    end_date: date | None = None             # None = till latest available NAV

    # Benchmark
    benchmark_index: str = "NIFTY 50 TRI"   # For comparison


@dataclass
class CashflowEvent:
    """Single investment event."""
    date: date
    amount: Decimal                          # Positive = investment
    nav: Decimal
    units: Decimal
    event_type: str                          # "SIP", "TOPUP", "LUMPSUM"
    trigger: str                             # "scheduled", "mild_correction", etc.


@dataclass
class DailySnapshot:
    """Portfolio state on one day."""
    date: date
    nav: Decimal
    cumulative_units: Decimal
    cumulative_invested: Decimal
    portfolio_value: Decimal
    # Benchmark tracking
    benchmark_value: Decimal | None = None


@dataclass
class SimulationResult:
    """Complete output of a simulation run."""
    params: SimulationParams
    mode: str

    # Summary statistics
    total_invested: Decimal
    final_value: Decimal
    absolute_return_pct: Decimal
    xirr_pct: Decimal | None
    cagr_pct: Decimal | None
    max_drawdown_pct: Decimal
    max_drawdown_start: date | None
    max_drawdown_end: date | None
    sharpe_ratio: Decimal | None
    sortino_ratio: Decimal | None

    # SIP-specific
    num_sips: int
    num_topups: int
    topup_invested: Decimal

    # Signal-specific
    signal_hit_rate_3m: Decimal | None     # % of top-ups that were profitable after 3 months
    signal_hit_rate_6m: Decimal | None
    signal_hit_rate_12m: Decimal | None
    capital_efficiency: Decimal | None      # Extra return per extra ₹ deployed via signals

    # Benchmark comparison
    benchmark_cagr_pct: Decimal | None
    alpha_vs_benchmark: Decimal | None

    # Time series (for charting)
    daily_timeline: list[DailySnapshot]
    cashflow_events: list[CashflowEvent]
    signal_events: list[SignalEvent]

    # Rolling metrics
    rolling_1y_xirr: list[dict]            # [{date, xirr}, ...] — monthly rolling 1Y XIRR
    monthly_returns: list[dict]             # [{month, return_pct}, ...]

    # Metadata
    fund_name: str
    mstar_id: str
    simulation_hash: str
    compute_time_ms: int


class SimulationEngine:
    """
    Core simulation engine. Pure computation — no DB access.
    """

    def simulate(
        self,
        params: SimulationParams,
        nav_series: list[tuple[date, Decimal]],   # [(date, nav), ...] sorted ascending
        signal_events: list[SignalEvent],           # Pre-computed signal events
        benchmark_series: list[tuple[date, Decimal]] | None = None,
        fund_name: str = "",
        mstar_id: str = "",
    ) -> SimulationResult:
        """
        Run a full simulation.

        Steps:
          1. Generate SIP dates (if applicable)
          2. Process SIP investments on each SIP date
          3. Process signal top-ups on each signal event date
          4. Process lumpsum deployment (if applicable)
          5. Build daily portfolio value timeline
          6. Compute XIRR from all cashflows
          7. Compute max drawdown from timeline
          8. Compute rolling returns
          9. Compute signal hit rates
          10. Compute benchmark comparison
          11. Package everything into SimulationResult
        """

    # --- Investment Processing ---

    def _generate_sip_dates(
        self, start: date, end: date, sip_day: int
    ) -> list[date]:
        """Generate SIP dates (Nth of every month in range)."""

    def _get_nav_on_date(
        self, nav_series: list[tuple[date, Decimal]], target: date
    ) -> tuple[date, Decimal]:
        """Get NAV on target date or next available trading day (up to 10 days forward)."""

    def _process_sip(
        self, sip_date: date, amount: Decimal, nav_series: list[tuple[date, Decimal]]
    ) -> CashflowEvent | None:
        """Process one SIP installment."""

    def _process_topup(
        self, event: SignalEvent, sip_amount: Decimal, nav_series: list[tuple[date, Decimal]]
    ) -> CashflowEvent | None:
        """Process one signal-triggered top-up. Amount = sip_amount × multiplier."""

    def _process_lumpsum(
        self, deploy_date: date, amount: Decimal, nav_series: list[tuple[date, Decimal]]
    ) -> CashflowEvent | None:
        """Process a lumpsum deployment."""

    # --- Statistics ---

    def _compute_xirr(
        self, cashflows: list[tuple[date, Decimal]]
    ) -> Decimal | None:
        """
        XIRR using Newton-Raphson method.
        Cashflows: [(date, -amount), ..., (date, +final_value)]
        Negative = money invested, positive = money returned.
        """

    def _compute_cagr(
        self, initial: Decimal, final: Decimal, years: Decimal
    ) -> Decimal | None:
        """CAGR = (final/initial)^(1/years) - 1"""

    def _compute_max_drawdown(
        self, timeline: list[DailySnapshot]
    ) -> tuple[Decimal, date | None, date | None]:
        """
        Max peak-to-trough drawdown.
        Returns (drawdown_pct, peak_date, trough_date).
        """

    def _compute_sharpe(
        self, monthly_returns: list[Decimal], risk_free_annual: Decimal = Decimal("0.06")
    ) -> Decimal | None:
        """Sharpe = (avg_monthly_return - rf_monthly) / std_monthly_return * sqrt(12)"""

    def _compute_sortino(
        self, monthly_returns: list[Decimal], risk_free_annual: Decimal = Decimal("0.06")
    ) -> Decimal | None:
        """Sortino = (avg_monthly_return - rf_monthly) / downside_std * sqrt(12)"""

    def _compute_rolling_xirr(
        self, cashflow_events: list[CashflowEvent], timeline: list[DailySnapshot]
    ) -> list[dict]:
        """
        Rolling 1Y XIRR computed monthly.
        For each month-end, compute XIRR using cashflows from the past 12 months.
        """

    def _compute_monthly_returns(
        self, timeline: list[DailySnapshot]
    ) -> list[dict]:
        """Monthly portfolio returns from daily timeline."""

    def _compute_signal_hit_rates(
        self, topup_events: list[CashflowEvent], nav_series: list[tuple[date, Decimal]]
    ) -> dict:
        """
        For each top-up event, check if the investment was profitable after 3m, 6m, 12m.
        Hit rate = profitable_topups / total_topups.
        """

    def _compute_capital_efficiency(
        self, signal_result: SimulationResult, baseline_result: SimulationResult
    ) -> Decimal | None:
        """
        How much extra return was generated per extra ₹ invested via signals.
        (signal_final - baseline_final) / (signal_invested - baseline_invested)
        """

    def _compute_simulation_hash(self, params: SimulationParams, mstar_id: str) -> str:
        """Deterministic hash for cache/dedup."""
```

**`test_signal_engine.py`** — Signal evaluation logic.

```python
# SignalCondition.evaluate
- BELOW: value < threshold → True
- ABOVE: value > threshold → True
- CROSSES_BELOW: previous above, current below → True
- CROSSES_BELOW: both below → False (not a crossing)
- CROSSES_ABOVE: previous below, current above → True

# SignalRule.evaluate
- AND with 2 conditions: both true → fires
- AND with 2 conditions: one false → doesn't fire
- OR with 2 conditions: one true → fires
- OR with 2 conditions: both false → doesn't fire

# SignalEngine.evaluate_range
- Date range with no signal data → empty events
- Date range where rule fires → SignalEvent created
- Cooloff respected: rule fires, then another trigger within cooloff → skipped
- Cooloff expired: rule fires, wait cooloff_days, fires again
- Multiple rules fire same day → highest multiplier wins
- Pre-built DEFAULT_RULES load without error
```

# SIP date generation
- 12-month range → 12 SIP dates on the Nth of each month

- Weekend → returns next Monday's NAV

- Signal events recorded in result

- Only 1 cashflow event
- CAGR = simple compound return

- SIP invested + lumpsum chunks on signals
- Reserve depleted as signals fire

# Max Drawdown
- Monotonically increasing NAV → drawdown = 0
- NAV drops 20% then recovers → drawdown = 20%, dates correct
- Multiple drawdowns → reports the worst one

- Single investment (lumpsum) → XIRR ≈ CAGR

- 24-month simulation → 12 rolling 1Y XIRR values (months 12-24)

- 12-month sim → 12 monthly return values

- Top-up followed by 10% gain in 3 months → hit
- Top-up followed by -5% in 3 months → miss
- Hit rate = hits / total

## Key Design Decisions

1. **Signal engine is separate from simulation engine.** SignalEngine produces events, SimulationEngine consumes them. You can test signals without running a simulation, and run simulations without signals (pure SIP/lumpsum modes).

2. **Default rules are pre-built but user-configurable.** The frontend rule builder lets users modify thresholds, add/remove conditions, change multipliers. Default rules are a starting point based on our experience with the MF Simulator.

3. **XIRR is the primary return metric, not CAGR.** XIRR handles irregular cashflows (SIP + top-ups). CAGR is only meaningful for lumpsum. Both are computed.

4. **MarketPulse unavailability = graceful degradation.** If MarketPulse is down, signal evaluation returns no events. SIP_SIGNAL mode degrades to pure SIP. The simulation still runs.

5. **Decimal for all money math.** Signal thresholds can be float (they're percentages, not money). NAV, amounts, returns, XIRR — all Decimal.

6. **Simulation hash for dedup.** Same params + same fund + same date range = same hash. Frontend can cache results.
