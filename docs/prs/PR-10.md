# PR-10: Simulation Lab — Signal Timeline + Rule Builder + 4-Mode Comparison

## Objective
Build the Simulation Lab page where users can backtest investment strategies across real NAV history using the 4-mode simulation engine (Pure SIP, SIP + Signal Top-ups, Pure Lumpsum, Hybrid). Features an interactive signal timeline, visual rule builder, and side-by-side mode comparison.

---

## APIs Consumed

- `POST /api/v1/simulation` — run simulation (single mode)
- `POST /api/v1/simulation/compare` — run all 4 modes simultaneously
- `GET /api/v1/simulation/rules/defaults` — default signal rules
- `POST /api/v1/simulation/validate-rules` — validate custom rules
- `GET /api/v1/funds/{mstar_id}` — fund detail for header
- `GET /api/v1/funds/{mstar_id}/nav?period=max` — full NAV history for timeline
- `GET /api/v1/lens/scores/{mstar_id}` — lens scores for fund context
- `GET /api/v1/marketpulse/breadth?lookback=1y` — breadth data for signal overlay
- `GET /api/v1/marketpulse/sentiment` — sentiment for signal overlay
- `GET /api/v1/funds?search=&limit=20` — fund search for picker

### File 2: `web/src/components/simulation/FundPicker.jsx`

Search-as-you-type fund selector. Uses `fetchFunds({ search: query, limit: 20 })`. Shows fund name, AMC, category, 1Y return in dropdown. Selecting a fund loads its data and starts the simulation flow.

### File 3: `web/src/components/simulation/RuleBuilder.jsx`

Visual signal rule builder — structured form, not drag-and-drop.

**Each rule is a card containing:**
- Rule name (editable text, e.g., "Breadth Panic")
- Conditions section: one or more condition rows
  - Each row: `[Signal Source ▾] [Operator ▾] [Threshold input]`
  - Signal sources: breadth_21ema, breadth_50ema, breadth_200ema, sentiment_composite, vix_level, nifty_vs_200sma
  - Operators: BELOW, ABOVE, CROSSES_BELOW, CROSSES_ABOVE
  - Threshold: numeric input
- Logic toggle: AND / OR (between conditions within the rule)
- Multiplier: dropdown (1x, 2x, 3x, 4x, 5x) — "deploy X times the SIP amount"
- Cooloff days: numeric input (default 30)
- Delete rule button (trash icon)

**Bottom controls:**
- "+ Add Rule" button (adds empty rule card)
- "Reset to Defaults" button (loads from `fetchDefaultRules()`)
- Rule validation: real-time feedback using `POST /simulation/validate-rules`

**On any rule change:** If auto-simulate is on, re-run simulation after 500ms debounce. Signal markers on the timeline update in real-time.

### File 4: `web/src/components/simulation/SimulationConfig.jsx`

Configuration panel above the rule builder:
- **SIP Amount:** ₹ input (default: ₹10,000/month)
- **Lumpsum Amount:** ₹ input (default: ₹5,00,000) — for lumpsum and hybrid modes
- **Lumpsum Deploy %:** Percentage of reserve per signal event (default: 25%)
- **SIP Date:** Day of month (default: 5th)
- **Auto-simulate toggle:** When on, any config/rule change triggers re-run

### File 5: `web/src/components/simulation/SignalTimeline.jsx`

D3.js interactive time-series chart. This is the hero visualization.

**Main panel (top, 70% height):**
- X-axis: time (dates). Y-axis: NAV value (₹)
- NAV line: solid teal-600, 2px stroke
- Signal event markers: vertical lines from x-axis to NAV line at each signal date
  - Breadth signals: red dots
  - Sentiment signals: amber dots
  - VIX signals: purple dots
  - Sector signals: teal dots
- Each marker has a small tooltip on hover: date, signal type, threshold hit, amount deployed

**Capital panel (bottom, 30% height, synced x-axis):**
- Step chart showing cumulative capital deployed over time
- Different colored areas for SIP contributions vs signal top-ups
- Y-axis: ₹ invested

**Interactions:**
- Zoom: scroll to zoom x-axis, drag to pan
- Hover: crosshair showing NAV, date, and any signals on that date
- Reset zoom button

**Real-time update:** When rules change in the RuleBuilder, signal markers appear/disappear with 300ms animation. Users can see "if I lower the breadth threshold from 40% to 35%, 3 more signals fire in 2022."

### File 6: `web/src/components/simulation/ModeComparison.jsx`

Side-by-side comparison of all 4 simulation modes.

**Result cards (4 columns):**
Each card shows:
- Mode name (Pure SIP / SIP+Signal / Lumpsum / Hybrid)
- XIRR (large, colored: green if > benchmark)
- Total invested
- Current value
- Absolute return
- Max drawdown (with red bar)
- Sharpe ratio
- Signal hit rate (only for modes with signals)
- Capital efficiency (return per unit of signal-deployed capital)

**Best mode highlighted** with a teal border and "Best" badge.

**Equity curves chart:**
Recharts LineChart with 4 lines (one per mode) overlaid. Colors: teal (Pure SIP), emerald (SIP+Signal), amber (Lumpsum), purple (Hybrid). Shared x-axis (time), y-axis (portfolio value ₹). Legend at bottom. This visually shows the gap — the signal-enhanced modes should visually diverge upward from pure SIP.
