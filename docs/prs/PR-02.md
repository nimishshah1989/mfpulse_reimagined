# PR-02: Fund Data Read APIs + Category Intelligence

## Objective
Build the read APIs that let the frontend (and any consumer) query fund data from the database. After this PR, you can search/filter/sort funds, get a full deep-dive for any fund, view category heatmaps, and compare funds side by side.

---

## Context

PR-01 built the ingestion pipeline that puts data INTO the database. PR-02 builds the APIs that get data OUT. These are pure read operations — no computation, no scoring. Just clean, fast, well-typed APIs for querying the Morningstar data we've ingested.

```python
class FundRepository:
    """Read operations for fund data. All queries name columns explicitly."""

    def __init__(self, db: Session): ...

    # --- Fund Master ---
    def get_all_funds(
        self,
        category: str | None = None,
        amc: str | None = None,
        search: str | None = None,      # Search by fund_name or legal_name
        purchase_mode: int | None = 1,   # Default: Regular only
        eligible_only: bool = True,
        limit: int = 100,
        offset: int = 0,
        sort_by: str = "fund_name",
        sort_dir: str = "asc",
    ) -> tuple[list[FundMaster], int]:
        """Returns (funds, total_count) for pagination."""

    def get_fund_by_mstar_id(self, mstar_id: str) -> FundMaster | None

    def get_fund_by_isin(self, isin: str) -> FundMaster | None

    def get_fund_by_amfi_code(self, amfi_code: str) -> FundMaster | None

    def get_categories(self) -> list[dict]:
        """
        Returns all distinct categories with fund counts.
        [{ "category_name": "Large Cap", "broad_category": "Equity", "fund_count": 42 }, ...]
        """

    def get_amcs(self) -> list[dict]:
        """
        Returns all distinct AMCs with fund counts.
        [{ "amc_name": "HDFC AMC", "fund_count": 38 }, ...]
        """

    # --- NAV & Returns ---
    def get_latest_nav(self, mstar_id: str) -> dict | None:
        """Latest NAV row for a fund."""

    def get_nav_history(
        self,
        mstar_id: str,
        start_date: date | None = None,
        end_date: date | None = None,
        limit: int | None = None,
    ) -> list[dict]:
        """NAV time series for charting. Returns date + nav + returns."""

        """Latest trailing returns across all periods (1D through 10Y)."""

    # --- Risk Stats ---
    def get_latest_risk_stats(self, mstar_id: str) -> dict | None:
        """Latest risk ratios for a fund."""

    def get_risk_stats_history(
        self,
        mstar_id: str,
        limit: int = 12,

    def get_latest_ranks(self, mstar_id: str) -> dict | None:
        """Latest quartile and absolute ranks."""

    # --- Category Returns ---
    def get_category_returns(self, category_code: str) -> dict | None:
```

```python
    # --- Overlap Analysis ---
    def get_holdings_for_overlap(
        self,
        mstar_ids: list[str],
    ) -> dict[str, list[dict]]:
        """
        Returns holdings for multiple funds keyed by mstar_id.
        Used by the overlap analyzer to find common stocks.
        """

    def compute_overlap(
        self,
        mstar_ids: list[str],
    ) -> dict:
        """
        Given 2-5 fund mstar_ids, compute:

        - common_holdings: holdings appearing in 2+ funds with combined weight
        - effective_allocation: aggregated sector/market cap if you held all funds equally
        """
```

```python
class FundSummary(BaseModel):
    """Fund list view — compact."""
    mstar_id: str

    category_name: str
    broad_category: str | None
    inception_date: date | None

    net_expense_ratio: Decimal | None
    # Latest data (joined from nav_daily)
    latest_nav: Decimal | None
    latest_nav_date: date | None
    return_1y: Decimal | None
    return_3y: Decimal | None
    return_5y: Decimal | None

class FundDetail(BaseModel):
    """Full fund deep-dive — everything we know."""
    # Master data
    fund: FundSummary
    # Risk profile
    indian_risk_level: str | None
    primary_benchmark: str | None
    investment_strategy: str | None
    managers: str | None
    # Latest returns (all periods)
    returns: dict | None
    # Latest risk stats
    risk_stats: dict | None
    # Latest ranks
    ranks: dict | None
    # Portfolio snapshot
    portfolio: dict | None
    # Top holdings
    top_holdings: list[dict]
    # Sector exposure
    sector_exposure: list[dict]
    # Asset allocation
    asset_allocation: dict | None
    # Credit quality (if applicable)
    credit_quality: dict | None
    # Category comparison
    category_avg_returns: dict | None
    # Peer rank context
    category_fund_count: int | None

class CategorySummary(BaseModel):
    """Category heatmap entry."""
    category_name: str
    broad_category: str | None
    fund_count: int
    # Average returns for the category
    avg_return_1y: Decimal | None
    avg_return_3y: Decimal | None
    avg_return_5y: Decimal | None

class NAVPoint(BaseModel):
    """Single NAV data point for charting."""
    date: date
    nav: Decimal
    return_1d: Decimal | None = None

class OverlapResult(BaseModel):
    """Result of overlap analysis."""
    funds_analyzed: list[str]
    overlap_matrix: dict          # {fund_a: {fund_b: overlap_pct}}
    common_holdings: list[dict]   # Holdings in 2+ funds
    effective_sector_allocation: list[dict]
    effective_market_cap: dict | None

class PeerComparison(BaseModel):
    """Fund vs category peers."""
    fund_mstar_id: str
    category_name: str
    peer_count: int
    fund_return_1y: Decimal | None
    category_avg_1y: Decimal | None
    fund_percentile_1y: int | None   # Where does this fund rank (0-100)
    peers: list[FundSummary]
```

## Key Design Decisions

1. **FundSummary includes latest NAV + returns** — the fund list endpoint joins fund_master with the latest nav_daily row so the table view shows current data without a second API call per fund.

2. **Period parameter for NAV history** — `1m`, `3m`, `6m`, `1y`, `3y`, `5y`, `max`. Compute start_date from period string. Don't return 10 years of daily data by default.

3. **Overlap analysis is a POST** — takes a request body with 2-5 mstar_ids. Returns pairwise overlap matrix + common holdings. Overlap is computed by matching holdings by ISIN (not by name, which can vary).

4. **All Decimal fields stay as Decimal** — Pydantic v2 serializes Decimal to string in JSON. This is correct for financial data. Frontend parses string → number for display.

5. **Pagination on list endpoints** — `limit` + `offset` with total count in meta. Default limit 100.

## Acceptance Criteria

1. `GET /api/v1/funds` returns paginated fund list with latest NAV data
2. `GET /api/v1/funds/{mstar_id}` returns complete fund deep-dive
3. `GET /api/v1/funds/{mstar_id}/nav?period=1y` returns NAV time series
4. `GET /api/v1/categories` returns all categories with fund counts
5. `POST /api/v1/holdings/overlap` computes overlap for 2+ funds
6. 404 returned for non-existent fund IDs
7. All queries use named columns (no SELECT *)
8. All tests pass
9. Response envelope consistent: `{ success, data, meta, error }`
